Instructions

Select the Begin Scavenger Hunt button to start learning. Follow the Instructions on each page. Instructions are marked *Instructions*. The instructions provide information on what tools to use and what to look for. The purpose of this scavenger hunt is to identify the web accessibility issues on each page, and correctly map them to their WCAG 2.0 Level A and AA success criteria. We have provided a template document that you can use to document the accessibility issues as they pertain to each WCAG success criteria. If you wish to have your final document reviewed, we are happy to provide feedback and a final score. Please submit your final report to ITaccessibility@ontario.ca. We recommend that you keep a physical or digital copy of the WCAG 2.0 Checklist with you. Each page in the scavenger hunt provides a clue as to which WCAG success criteria the issue should be mapped to.

Note: The accessibility issues as found in the scavenger hunt are by no means comprehensive. The purpose is to provide a better connection between the WCAG success criteria and the tools used for accessibility analysis, encouraging more comfort with reporting.
