Instructions:

  1. If you're currently using Internet Explorer, open a new Chrome browser window and copy the URL in the address bar. Paste the URL in your newly opened Chrome browser and use Chrome to test this page.
  2. Review the content on the page using the keyboard Tab key. Focus on the area labeled, Area of Issue.
  3. Review your WCAG 2.0 Checklist, success criteria 2.1.2 No Keyboard Trap.
  4. Determine the accessibility issue with the content on this page.
  5. Document the issue in the WCAG Accessibility Compliance Report Excel Spreadsheet.
  6. Once the issue has been documented, please proceed to the next page by selecting the Next Page button.

Area of Issue

A keyboard trap occurs when a user can get into a component or element on a web page by using the keyboard but cannot get out of that component or element through the use of the keyboard.



Tab to the next focusable element

This is an issue because not everyone who uses the internet will use a mouse for navigation. This may be due to personal preference, limitations of their hardware or some form of disability that prevents them from being able to use a mouse effectively.
